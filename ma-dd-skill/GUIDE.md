# M&A 财务尽调 Skill — 用户指南

## 这是什么

一套覆盖 M&A 财务尽调全流程的 AI 分析框架：从尽调报告解读、风险提取，到财务建模、估值分析，再到投资备忘录草稿。**估值环节采用达摩达兰（Aswath Damodaran, NYU Stern）的估值思维结构**——不是机械套公式，而是"故事→数字→故事"的专业判断框架。

## 7 大核心能力

| 能力 | 说明 | 优先级 |
|------|------|--------|
| **1. 报告解读与风险提取** | 从 FDD/Tax DD 报告中提取结构化风险清单、EBITDA 调整桥、净负债分析、红旗警示 | ⭐⭐⭐ 核心 |
| **2. 财务建模** | 基于尽调数据搭建 5 年三表预测（P&L + BS + CF），含三情景分析 | ⭐⭐⭐ 核心 |
| **3. 估值分析（达摩达兰驱动）** | DCF + EBITDA 倍数双重估值，含敏感性分析、估值三角验证、情景概率加权 | ⭐⭐⭐ 核心 |
| **4. 税务尽调风险梳理** | 税务风险登记表、税务属性分析、交易结构影响、SPA 保护建议 | ⭐⭐ |
| **5. 投资备忘录草稿** | 完整的董事会级投资备忘录，含执行摘要、投资逻辑、风险评估、谈判策略 | ⭐⭐ |
| **6. HTML 报告生成** | 整合所有分析产出为麦肯锡风格 HTML 报告（含 4 张 SVG 图表），零外部依赖、立即可打印 | ⭐⭐ |
| **7. 达莫达兰归纳附录** | 完整的第一人称 7 维度估值哲学论述，作为独立附录呈现 | ⭐ |

## 适用人群

- **CFO / 财务总监**：需要快速消化第三方尽调报告，形成内部决策依据
- **投资总监 / M&A 负责人**：需要独立估值意见，交叉验证 FA 给的估值
- **企业战略/战投团队**：需要标准化的交易分析框架
- **PE/VC 投资团队**：需要快速的财务尽调审阅和第二估值意见

## 如何使用

### 准备工作

准备以下材料（越多越好）：

**理想情况（完整尽调）：**
- 财务尽调报告（FDD report）
- 税务尽调报告（Tax DD report）
- 目标公司历史财务报表（3-5 年）
- 管理层财务预测（如有）

**最小配置（快速估值）：**
- 目标公司基本信息（行业、业务模式）
- 历史财务数据（至少 2 年的 P&L + 最近一期 BS）
- 交易背景（为什么要买、买多少、大概多少钱）

### 部署方式

**Claude Projects（推荐）：**
1. 打开 claude.ai，创建新 Project
2. 将 `PROMPT.md` 的**完整内容**粘贴到 Project instructions（系统指令）
3. 将 `damodaran-distill.md`、`valuation-templates.md`、`output-templates.md`、`mckinsey-report.md`、`report-template.html` 上传到 Project knowledge（知识库文件）
4. 保存，开启新对话

**ChatGPT GPTs：**
1. 创建新 GPT
2. 将 `PROMPT.md` 的完整内容粘贴为 Instructions
3. 将其他 5 个文件上传为 Knowledge 文件
4. 保存并发布（可设为仅自己可见）

### 对话流程

Skill 有 8 个 Phase（0-8），AI 会按顺序引导你：

```
Phase 0: 语言确认（一句确认即可）
    ↓
Phase 1: 交易背景（一次性回答 7 个问题，约 2 分钟）
    ↓
Phase 2: 粘贴报告内容 / 财务数据
    ↓
Phase 3: AI 输出风险清单 + 红旗警示（核心交付物 #1）
    ↓  [你审阅确认后继续]
Phase 4: AI 输出 5 年财务模型（核心交付物 #2）
    ↓  [你审阅假设后继续]
Phase 5: AI 以达摩达兰框架做估值分析（核心交付物 #3）
    ↓  [最需要你参与的环节——讨论假设、调整参数]
Phase 6: 税务风险摘要（如有 Tax DD）
    ↓
Phase 7: 投资备忘录草稿（核心交付物 #4）
    ↓
Phase 8: McKinsey 风格 HTML 报告生成（含 4 张 SVG 图表）
```

**你不必走完所有 Phase。** 可以：
- 只做 Phase 3（快速审阅尽调）
- 只做 Phase 5（独立估值意见——直接用达摩达兰框架）
- 只做 Phase 7（已有分析，仅需备忘录）

## 各交付物说明

### 交付物 1：风险清单（Phase 3）

包含：
- **EBITDA 调整桥**：从报告 EBITDA 到调整后 EBITDA，每项调整有分类和来源
- **净负债明细**：标准负债 + 类负债项目（debt-like items）
- **标准化 NWC 分析**：历史 NWC 趋势 + 推荐 peg
- **风险登记表**：按财务/税务/运营/法律分类，红黄绿三色评级
- **红旗警示 Top 5-10**：区分"交易破坏者"和"价格调整项"

### 交付物 2：财务模型（Phase 4）

包含：
- 5 年 P&L 预测（含历史对比列）
- 5 年 BS 预测
- 5 年 CF 预测
- 三情景分析（乐观/基准/悲观——每个情景都有故事，不是随便 ±20%）
- CSV 导出块（可复制粘贴到 Excel）

### 交付物 3：估值分析（Phase 5）⭐ 达摩达兰驱动

这是本 skill 最独特的环节。AI 会先讲公司故事，再做数字分析：

- **公司故事段落**（先叙事，后数字）
- WACC 计算（含每个参数的理由）
- DCF 估值（两种终值方法 + 终值占比警戒检查 + 增长率质疑测试）
- 敏感性分析表（WACC × 终值增长率）
- EBITDA 倍数估值（**含倍数解压**——你选的倍数隐含着什么增长/风险/回报假设？）
- 估值三角验证 + 橄榄球图
- 情景概率加权估值
- **一页纸估值总结**（投资委员会级别）

### 交付物 4：投资备忘录（Phase 7）

完整的董事会级投资备忘录草稿：
1. 执行摘要
2. 目标公司概览
3. 投资逻辑与协同估算
4. 财务分析摘要
5. 估值分析
6. 风险评估
7. 谈判策略与底线
8. 建议（Go / Conditional Go / No-Go）

### 交付物 5：McKinsey 风格 HTML 报告（Phase 8）⭐ New

将所有分析整合为一份独立的 HTML 报告文件：
- 封面 → SCQA 执行摘要 → 8 个 Exhibit → 5 个附录
- 4 张内联 SVG 图表：EBITDA 瀑布图 / GP 趋势 / Football Field / 风险矩阵
- 附有完整达莫达兰第一人称估值归纳
- 零外部依赖，可直接在浏览器打开或打印为 PDF

## 达摩达兰估值模式的独特之处

本 skill 的 Phase 5 与传统"输入参数、出结果"的估值工具有本质不同：

| 传统估值工具 | 本 skill（达摩达兰模式） |
|-------------|----------------------|
| 直接让你填 WACC、增长率 | 先让你讲公司故事，再讨论参数 |
| 给你一个数字 | 给你一个区间 + 每种情景的概率 + 概率加权结果 |
| "假设永续增长 3%" | "这个 g 合理吗？你的 g 不能超过经济增速" |
| "行业倍数 10x，就用这个" | "10x 隐含了什么增长率和 ROIC？和你的 DCF 一致吗？" |
| 终值占 85% 也不提醒 | 终值 >70% 立刻标黄警告 |
| 输出即结束 | 必须给你一页纸讲清楚"为什么值这个钱" |

## 数据隐私说明

- 在 Claude Projects / ChatGPT 中使用时，对话数据的使用遵循各平台政策
- **建议**：对目标公司名称、敏感数据进行脱敏处理（如"目标公司"、"Company X"）
- **如果涉及实际交易**：建议使用企业级部署方案，确认数据处理条款符合 NDA 要求
- 本 skill 本身不收集、不存储、不传输任何数据——它是纯 prompt 模板

## 局限性（重要！）

1. **AI 输出需要专业人士审阅**：本 skill 提供的是分析草稿和判断框架，不是最终决策依据。所有估值假设、风险判断、法律条款建议都需要持牌专业人士确认。

2. **尽调报告文本质量影响输出质量**：如果尽调报告是扫描版、格式混乱、数据缺失，AI 的提取和分析会受影响。Garbage in, garbage out.

3. **估值依赖于输入假设的质量**：AI 会质疑你的假设，但如果基础数据是错误的，估值结果也会是错误的。AI 不做事实核查——它只能基于你提供的信息进行推理。

4. **不是法律建议**：SPA 条款建议、税务结构分析仅供参考，不构成法律或税务意见。

5. **不替代外部顾问**：AI 不能替代会计师事务所、律师事务所、FA 的专业服务。它是你消化和交叉验证外部顾问工作的工具，不是外部顾问的替代品。

## 常见问题

**Q: 没有尽调报告，只有历史财务报表，能用吗？**
A: 可以。Phase 3（风险提取）会比较薄弱，但 Phase 4（建模）和 Phase 5（估值）完全可以基于历史数据 + 你的行业判断来做。

**Q: 我只想要估值分析，不要其他的，可以吗？**
A: 可以。直接说"我只需要估值分析"，AI 会跳到 Phase 1（简单了解背景）→ Phase 2（提取历史数据）→ Phase 5（达摩达兰估值）。

**Q: 达摩达兰模式太啰嗦了，能不能跳过故事直接给估值？**
A: 技术上可以，但强烈不建议。达摩达兰整个方法论的核心就是"没有故事的数字是猜测"。如果你真的赶时间，可以说"快速估值模式"——AI 会简化流程但保留核心检查（增长率质疑、终值检查、倍数解压）。

**Q: Excel 模型文件怎么输出？**
A: AI 不能生成 .xlsx 文件。但每个财务表格都有配套的 CSV 代码块，可以复制粘贴到 Excel 中。对于复杂模型，AI 会提供公式和单元格逻辑描述。
